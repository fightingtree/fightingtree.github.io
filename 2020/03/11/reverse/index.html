系统引导过程

系统引导与恶意软件的关系

恶意软件如何再次获得控制权：

  • 自身被结束之后
  • 操作系统重启之后
  • 操作系统重装之后
  • 硬盘更换之后

BIOS

BIOS(Basic Input and Output System,基本输入输出系统)，存储在主板BIOS Flash(或ROM)芯片上，为计算机提供最底层的、最直接的硬件设置和控制。计算机重启或开机之后，首先是BIOS获得计算机控制权，再与计算机操作系统进行对接。

BIOS引导模式

  1. 传统BIOS模式
  2. UEFI(Unified Extensible Firmware Interface,统一可扩展固件接口)
两种模式下的启动流程
  1. 传统BIOS引导：开机->BIOS引导->POST自检->MBR(硬盘第一个扇区，主引导扇区，硬盘主引导程序)->DBR(活动分区引导程序)->bootmgr(或NTLDR，操作系统引导)->读取BCD(或boot.ini)->启动对应的系统
  2. UEFI BIOS引导：开机->BIOS初始化->启动ESP分区管理器(EFI分区bootxxx.efi)->加载BCD->调用对应系统盘winload.efi->启动对应系统

BIOS的自检与初始化工作

任务：检测系统中的一些关键设备(如内存或显卡)是否存在和能否正常工作，进行初始化，并将控制权交给后续引导程序。

  • 可以根据用户指定的顺序从软盘、硬盘或光驱启动。如果从硬盘启动，则将控制权交给硬盘主引导程序。

硬盘主引导程序

MBR(Master Boot Record)，硬盘第一个分区

主要功能

  • 通过主分区表定位活动分区
  • 装载活动分区的引导程序，并移交控制权

活动分区引导程序

DBR(DOS Boot Record),或称OBR(OS Boot Record),或称分区引导记录(PBR,Partition Boot Record),分区的第一个扇区

功能

加载操作系统引导程序(如Windows XP系统的那NTLDR)

操作系统引导

以Winodws NTLDR为例：

  1. 将处理器从16位内存模式扩展为32(64)位内存模式
  2. 启动小型文件系统驱动，以识别FAT32和NTFS文件系统
  3. 读取boot.ini，进行多操作系统选择(或hiberfil.sys恢复休眠)
  4. 检测和配置硬件
系统内核引导
  1. NTLDR加载内核程序NTOSKRNL.EXE以及硬件抽象层HAL.dll等
  2. 读取并加载HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet下指定的驱动程序
  3. NTLDR将把控制权传递给NTOSKRNL.EXE，至此引导过程结束。

Windows系统装载

  1. 创建系统环境变量
  2. 启动Win32.sys(Windows子系统的内核模式部分)
  3. 启动csrss.exe(Windows子系统的用户模式部分)
  4. 启动winlogon.exe
    屏幕显示：Windows logo界面和进度条

登录阶段

  1. 启动需要自动启动的Windows服务
  2. 启动本地安全认证Lsass.exe
  3. 显示登录界面

Windows登录之后

系统启动当前用户环境下的自启动项程序

  • 注册表特定键值
  • 特定目录(如startup)等
    用户触发和执行各类程序

硬盘

硬盘是控制代码的静态存储仓库，恶意软件争夺控制权的中心。

硬盘的结构

  • 主引导扇区
  • 基本分区
  • 扩展分区

硬盘的逻辑结构

主引导扇区

位于整个硬盘的0柱面0磁头1扇区(硬盘的第一个扇区)

  • MBR引导程序：占了前446个字节(偏移0H-1BDH)
  • DPT(硬盘分区表)：随后的64字节
  • 结束标志：最后的两个字节55AA(便宜1FEH-1FFH)
