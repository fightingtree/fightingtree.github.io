使用github的目的

使用github托管项目

基本概念

仓库Repository

即项目，若要在github中开源一个项目，就必须新建一个repository，一个repository就是一个项目

收藏Star

其他用户收藏你的项目的人数

复制克隆项目Fork

其他用户可以以你开源的项目为基础，随心所欲进行改进并且加以运用，但不会影响原有项目的代码和结构

发起请求Pull Request

基于fork的操作，其他用户对开源项目进行改进后，可以原有项目的创建人发起pull request(简称PR)将他所做的的改进加入原有项目中

关注Watch

事务卡片Issue

其他用户可以通过issue向项目创建人提出问题和建议
